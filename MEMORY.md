# Memory Index — AI 内容分析助手

## 用户偏好
- 语言偏好：报告和分析内容使用中文输出
- 技能优先：优先使用 `skill-content-crawler` 进行多平台爆款内容采集与分析
- 报告风格：偏好结构化报告（分平台拆解 + 对比分析 + 实操建议 + 变现路径）
- 执行要求：拒绝现成报告，必须实际运行爬虫采集真实数据
- 完成提醒：多平台采集全部成功后需要主动提醒用户
- 进度汇报：用户会主动询问进度，需清晰展示各平台完成状态
- 登录流程：抖音/小红书采用浏览器自动化方案 A（弹窗→用户扫码→自动缓存 Cookie）
- 数据完整性：三平台（B 站/抖音/小红书）数据齐全后才生成最终完整版报告
- 本地执行：沙箱环境无法显示浏览器窗口时，用户选择在本地运行脚本完成扫码登录

## 项目约定
- 爆款分析维度：标题公式、内容结构、视觉呈现、发布策略、变现方式
- 目标平台：小红书、抖音、B 站为核心分析对象
- 选题方向：AI工具、效率提升、自媒体运营、内容创作、办公硬件（如钉钉A1）
- B 站 API：中文关键词需 URL 编码（`urllib.parse.quote`）
- Cookie 机制：抖音/小红书需浏览器自动化登录，Cookie 缓存于 `.cookie_cache` 目录
- 统一入口：通过 `scripts/run.py` 配置 PLATFORMS 和 KEYWORDS 控制多平台采集
- 跨平台报告：双平台数据可生成初步对比报告，三平台齐全后生成完整版
- 小红书特性：100% 图文笔记为主，收藏率是关键指标（平均收藏/点赞比 1.3:1）
- 数据迁移：同赛道（AI工具/效率科技）数据可用于新选题参考，爆款逻辑高度相通

## 注意事项
- 爬虫依赖：`undetected-chromedriver`、`selenium`、`whisper`、`ffmpeg` 需预先安装
- Python 3.12 兼容性：`undetected_chromedriver` 需要 `setuptools` 提供 `distutils` 模块
- Cookie 有效期：各平台 Cookie 有效期约 30 天，过期需重新登录获取
- 网络超时：B 站字幕提取可能因网络超时失败，需重试或跳过；小红书 ChromeDriver 下载易受网络影响
- 单平台模式：可单独运行某平台脚本避免依赖冲突
- 报告生成：抖音报告文件名可能误标为"bili_report"，需手动校验内容来源
- 小红书采集：部分笔记可能因反爬机制无法提取正文，属正常现象（成功率约 80%）
- 沙箱限制：execute_shell 在 bash 环境中运行，不支持 Windows 批处理和带引号的复杂命令
- 环境修复：需在沙箱 Python 环境安装 `undetected-chromedriver` 和 `setuptools`
- 本地运行：用户选择方案 A 在本地环境运行脚本完成浏览器扫码登录
